# Skill: Documentation

## Purpose
Write and maintain project documentation.

## Steps
1. **Read existing docs** — understand the project's voice, structure, and conventions.
2. **Identify gaps** — what is undocumented? What is outdated?
3. **Draft** — write clear, concise prose. Use active voice. One idea per paragraph.
4. **Add examples** — every API endpoint, every CLI command, every config option gets an example.
5. **Review formatting** — proper Markdown, working links, valid frontmatter.

## Constraints
- Never edit source code files (`.rs`, `.ts`, `.py`, `.toml`).
- Follow the project's heading hierarchy.
- Keep ADRs under 200 lines.

## Output Format
- Changed file paths
- Diff summary
- Confidence score based on: accuracy, completeness, style consistency
