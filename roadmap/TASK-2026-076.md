## Context

Add a guided first-run wizard that replaces the bare "Open project folder" landing with a multi-step onboarding flow.

## Acceptance Criteria

- [x] First launch shows onboarding, not just the project picker
- [x] User can open an existing valid Orqestra repo through the wizard
- [x] User can generate and open a sample project through the wizard
- [x] Missing AI/cloud setup does not block local PM use
- [x] Onboarding completion persists across app restart
- [x] User can reopen setup/readiness after onboarding

## Implementation

- `commands/onboarding.rs` — state persistence with `OnboardingStateManager`
- `onboarding/OnboardingWizard.tsx` — 3-step wizard (welcome, project, readiness)
- `onboarding/WelcomeStep.tsx` — product intro with 3 action buttons
- `onboarding/ProjectStep.tsx` — folder picker + sample project creation
- `onboarding/ReadinessStep.tsx` — environment status display
- `App.tsx` — checks onboarding state on mount, shows wizard if not completed
