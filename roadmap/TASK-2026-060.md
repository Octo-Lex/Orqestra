## Context
The `git-bridge` crate currently shells out to the `git` CLI for all operations (commit, diff, status, pull, push). This is fragile and slow. Replace with the `gix` crate for native Rust git operations — in-process index manipulation, tree building, and commit creation without spawning a subprocess.

## Acceptance Criteria
- [ ] `git-bridge` uses `gix` for commit, diff, status operations
- [ ] No `std::process::Command` calls remain in git-bridge
- [ ] All 10 existing tests pass without modification
- [ ] Semantic commit pipeline works end-to-end with native gix
- [ ] Benchmark: commit creation <50ms (vs ~200ms with CLI shelling)
