## Summary
Stronghold plugin preflight showed JS-invocation-only API incompatible with Rust-side access.
Used `keyring-core` + `windows-native-keyring-store` as the approved spec fallback.
SecretVault trait is the release contract; OS keychain is the encrypted vault.
Legacy XOR migration with safe-verify-then-delete pattern.
Token masking for all errors crossing the Rust/TS boundary.
