## Context
Orqestra projects need automated dependency vulnerability scanning. The engine should parse `Cargo.lock`, `package-lock.json`, and `requirements.txt`, check against OSV/NVD databases, and surface findings as advisory tasks in the roadmap.

## Acceptance Criteria
- [ ] Parse `Cargo.lock` for Rust crate versions
- [ ] Parse `package-lock.json` for npm package versions
- [ ] Parse `requirements.txt`/`pyproject.toml` for Python dependencies
- [ ] Check against OSV API for known vulnerabilities
- [ ] Surface findings as advisory tasks with severity ratings
- [ ] Integration with ConfidenceGate for auto-flagging critical vulns
