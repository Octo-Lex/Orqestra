# v1.8.0 Tree-sitter Code Intelligence & Symbol Context Evidence

## Purpose

Add symbol-level code intelligence so bugfix proposals and semantic commit preparation can reason about affected functions, classes, modules, imports, and public APIs.

## New Crate

`crates/code-intel/` — pure Rust, zero Tauri/git-bridge dependency.

### Dependencies
- `tree-sitter = "0.24"`
- `tree-sitter-rust = "0.23"`
- `tree-sitter-typescript = "0.23"`
- `serde = "1"`

## Language Support

| Language | Extensions | Symbols Extracted |
|----------|-----------|-------------------|
| Rust | .rs | functions, methods, structs, enums, traits, impls, type aliases, modules, imports, constants |
| TypeScript | .ts, .tsx, .js, .jsx | functions, classes, interfaces, enums, type aliases, methods, imports, arrow functions |

## Exclusion Rules

| Category | Examples | Status |
|----------|----------|--------|
| Generated/vendor dirs | target/, node_modules/, dist/, build/, .git/, .Orqestra/ | Excluded |
| Secret-risk paths | .env, *.pem, id_rsa | Secret |
| Binary files | .png, .exe, .zip, .pdf | Binary |
| Large files | >256 KiB | TooLarge |
| Unsupported languages | .md, .yaml, .toml | Excluded |

## Parse Error Detection

Tree-sitter returns trees even for malformed code. ParseStatus::ParseError means:
- Parser returned no tree, OR
- ERROR/MISSING node ratio exceeds 30%

## Deterministic Ordering

Symbols sorted by: line_start → line_end → kind → name → parent

## Architecture

```
code-intel (pure Rust, no deps on Tauri or git-bridge)
  ├── lib.rs — DTOs, public API, language detection, deterministic sorting
  ├── extract.rs — tree-sitter Rust/TS extraction with safe field access
  └── exclude.rs — file exclusion rules (duplicated locally, not imported from git-bridge)

orqestra-desktop
  └── commands/code_intel.rs — Tauri commands (separate from git.rs)
```

## Circular Dependency Guard

```
code-intel → (no dependency on git-bridge)
git-bridge → (may call code-intel)
```

## Agent Wiring

- Docs-agent: symbol context disabled by default
- Semantic commit prep: references file-level affected symbols (not hunk-level)
