## Attempt 3: Xvfb

Xvfb failed to start due to WSL2 filesystem restriction:
```
_XSERVTransSocketCreateListener: failed to bind listener
_XSERVTransSocketUNIXCreateListener: ...SocketCreateListener() failed
chmod: changing permissions of '/tmp/.X11-unix': Read-only file system
```

WSL2's `/tmp/.X11-unix` is on a read-only filesystem layer, preventing Xvfb from creating its Unix domain socket.

## Classification

**`webview-runtime-failure`** (specific: GTK initialization failure)

The Tauri/tao Linux event loop requires a functioning GTK display server. Without X11 or Wayland, the app panics during GTK init in `tao-0.35.3`. This is expected behavior — Tauri desktop apps need a display server.
